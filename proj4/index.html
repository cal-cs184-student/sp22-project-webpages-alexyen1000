Cloth Sim

Overview

In this project, we implemented the logic necessary to realistically simulate a cloth’s movement using masses and springs. This involved applying various physics equations to model the spring’s motion with respect to gravity and forces. We also implemented logic to allow the cloth to realistically collide with other objects, such as a sphere and a plane, in addition to the logic necessary for the cloth to self-collide (fall on itself). Finally, we implemented various shading models, such as diffuse shading, blinn-phong shading, texture mapping, displacement and bump mapping, and mirror shading (from an environment map).

Part 1: Masses and Springs

In this part we build the grid of point masses and springs, as seen in the images below. To do this, we iterate through the desired grid space using a nested for loop. We define (x’, y’) index of each point to be the top left corner of each “cell” in the grid. Note that this is different from the (x, y, z) coordinate of the point in 3D space.

Depending on whether the orientation is horizontal or vertical, we assign the (x, y, z) coordinates of the point mass to be different values. If it’s horizontal, x = x’, y = 1, and z = y’. If it’s vertical, x = x’, y = y’, and z is equal to a random value between -1/1000 and 1/1000. For each point mass, we also have to check if the (x’, y’) index is in the list of pinned points. If it is, when we add the point mass to the vector of point masses, we also specify that this point’s pinned status is true.

Next, we move on to creating the springs. There are 3 types of springs, Structural, Bending, and Shearing. Structural springs are ones that connect a point mass with another to its immediate left and top. Bending springs connect a point mass with the two springs that are two away to the left and top. Shearing springs are those that connect it to the point masses to the top left and top right. We iterate through every point mass that we’ve added to the vector, and for every spring at all 6 possible locations, we add the corresponding type to the vector of springs if the point mass at that location exists

Part 3: Handling Collisions with Other Objects

We implemented sphere and plane collisions in this part. For both kinds of collisions, we first determined whether or not there was a collision. If there was, we computed the correction vector and applied it to the point mass’s position such that the point mass’s position would be placed at the surface of the sphere/plane. We ran into an issue in the simulate() function where we were creating new point masses instead of updating the existing ones in the for loop. Once we figured out that we needed to write (auto& point_mass) instead of (auto point_mass), our implementation worked.

Part 4: Handling Self Collisions

In this part, we implemented the functions hash_position, build_spatial_map, and self_collide in order to effectively account for self-collisions. The hash_position function works by computing a unique key from the vector’s position coordinates x, y, and z. We followed the recommended approach of partitioning the space into boxes by utilizing w, h, and t and converting them into a unique hash. Our build_spatial_map function works by populating the hash map with the point masses, and creating an empty vector if needed. Our self-collide function works by first determining whether or not there is a collision. If there is, we compute the correction vector and apply it to the point mass’s position to successfully simulate self-collision. One issue we ran into while writing the hash_position function was computing the unique hash efficiently. We tried different methods which ended up with a hash table that had many collisions but eventually settled on leveraging prime numbers which helped minimize collisions.

When the density is low, the mass of the cloth is low, and thus all parts of the cloth seem to fall in a more “solid” way. As we increase the ks, the springs become stiffer and thus we see more of a “solid” falling behavior. As ks becomes smaller, the springs become looser, and thus we see the cloth fall with fine granularity.

Mirror Shading

To implement mirror shading, we first calculated w_o, the outgoing ray, by subtracting the fragment’s position from the camera’s position. We then reflected this ray on the surface normal and multiplied by -1 to get w_i’s incoming direction. Finally, we sampled the texture at this w_i to get the final texture color.
