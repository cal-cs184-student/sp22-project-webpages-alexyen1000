Overview

In this project, my partner and I decided to implement Part 1 and Part 4. Part 1 consisted of implementing the logic required to be able to render glass and mirror models. Meanwhile, Part 4 consisted of implementing the logic necessary for objects to have a depth of field effect, while is how cameras actually work in real life. This effect is achieved by simulating a thin lens, where only objects on a certain plane/within a certain distance are in focus. Up to this point, we had been using a pin-hole camera model where all objects are always in focus. Implementing these two parts gave us a more in-depth understanding of how models that simulate different materials are implemented (Part 1), and how real cameras can be simulated in code (Part 4).

We ran into a little snag with Part 4 with images rendering something that looked off to us, but it turned out that we needed to play around with the aperture settings of the camera.

Part 1: Mirror and Glass Materials

To do this, we essentially followed the instructions laid out in spec. First we made the reflect function, which flipped the direction of the wo ray and assigned that value to the wi pointer. Then we filled out the sample_f function with the relevant pdf and called reflect(). We return reflectance / abs_cos_theta(*wi)

For refracting, we calculate eta depending on if we’re entering or leaving the surface, and then assign the x/y/z values of wi to their correct values. We normalize and return true if there are no internal refractions. We also fill out the sample_f function for refracting, doing something similar to the sample_f with reflectance, and return transmittance / abs_cos_theta(*wi) / eta^2

Finally, to render glass objects which have both reflecting and refracting properties, we follow the pseudocode in spec. To calculate R using the Schlick’s approximation, we set n2 to be the objects index of refraction, and n1 to be 1 since the index of refraction of air is approximated to that.

Part 4: Depth of Field

The main difference between the pinhole camera model and the thin-lens camera model is that all objects are in focus in the pinhole camera model. By contrast, in the thin-lens camera model, only objects in a certain plane at a certain distance are in focus.

To implement this, we first took the code from 3-1’s generate_ray, allowing us to generate a ray through the center of the lens. To calculate pFocus, we used the ray-intersection formula to calculate t, using the point (0, 0, -focalDistance) to act as a point in the focal plane. Knowing t, we’re now able to call Ray::at_time(t) to find pFocus. We just have to subtract pLens, which is calculated as seen in spec, and then transform it using c2w to get a direction. Assigning those values to our new ray (pLens + pos) and the new direction is our output.

Results: Changing the focal length varies the portion of the object that’s in focus, with the best looking image (in terms of depth of field) at focal length = 4.5. Increasing the aperture increases the blurriness of the object, and decreases the portion of the object that’s in focus. The best looking image (in terms of depth of field) was at aperture = 0.25.
